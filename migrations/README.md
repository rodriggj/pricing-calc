# Migrations

SQL migrations generated by `drizzle-kit generate:pg` and applied by the
programmatic migrator in `scripts/migrate.ts`.

## Running Migrations

```bash
# Set DATABASE_URL (or ensure .env.local is loaded by your shell)
export DATABASE_URL=postgresql://estimating_app:dev_only_password@localhost:5432/estimating_app

pnpm db:migrate
```

The migrator reads `DATABASE_URL` from the environment. See `.env.local.example`
for the local development connection string.

## Generating New Migrations

After modifying files in `src/db/schema/`:

```bash
pnpm db:generate
```

This produces a new SQL file in this directory. **Hand-inspect** the generated
SQL before committing — drizzle-kit 0.20.x does not emit:

- `CHECK` constraints (expressed via `check()` in the schema)
- `WHERE` clauses on partial unique indexes

These must be added manually to the generated SQL.

## Idempotency

Running `pnpm db:migrate` against an already-migrated database is safe — the
migrator tracks applied migrations in a `__drizzle_migrations` journal table and
skips those already applied.
